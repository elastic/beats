[[new-features-8.19.14]]
=== New features

The 8.19.14 release adds the following new and notable features.




**Filebeat**

* Add retry back-off logic to streaming input CrowdStrike follower. https://github.com/elastic/beats/pull/49768[#49768] https://github.com/elastic/beats/pull/49896[#49896] https://github.com/elastic/beats/pull/49898[#49898] https://github.com/elastic/beats/pull/49899[#49899] https://github.com/elastic/beats/pull/49920[#49920] 
* Add secret_state config to CEL input for encrypted storage of secrets accessible as state.secret. https://github.com/elastic/beats/pull/49768[#49768] https://github.com/elastic/beats/pull/49896[#49896] https://github.com/elastic/beats/pull/49898[#49898] https://github.com/elastic/beats/pull/49899[#49899] https://github.com/elastic/beats/pull/49920[#49920] 
+
Add a secret_state configuration field to the CEL input. When configured in a
Fleet integration package with secret: true, the values are stored encrypted by
Fleet. At runtime, the contents are placed at state.secret and unconditionally
redacted in debug logs. The key &#34;secret&#34; in the plain-text state configuration
is reserved and rejected by validation to prevent accidental unencrypted storage
of values intended to be secret.



**Metricbeat**

* Add client secret authentication support to Azure App Insights module. https://github.com/elastic/beats/pull/49768[#49768] https://github.com/elastic/beats/pull/49896[#49896] https://github.com/elastic/beats/pull/49898[#49898] https://github.com/elastic/beats/pull/49899[#49899] https://github.com/elastic/beats/pull/49920[#49920]
